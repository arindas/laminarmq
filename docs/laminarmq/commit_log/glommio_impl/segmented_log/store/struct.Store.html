crate::commit_log::segmented_log::store::Store implementation for the [glommio] runtime.

This implementation uses directly mapped files to leverage io_uring powered operations. A Store consists of a [DmaFile] reader and [DmaStreamWriter], both of which point to the same underlying file on the disk.

We have chosen a simple [DmaFile] for the reader since the read amplification caused due to random unbuffered reads is manageable. However, write amplifications are generally more costly. Since we always append to the end of the file, [DmaStreamWriter] is a better solution which enables use to have the best tradeoff of responsiveness v/s buffering.
