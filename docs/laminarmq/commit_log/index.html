Module providing a “commit log” abstraction.

This kind of data structure is primarily useful for storing a series of “events”. This module provides the abstractions necessary for manipulating this data structure asynchronously on persistent storage media. The abstractions provided by this module are generic over different async runtimes and storage media. Hence user’s will need to choose their specializations and implementations of these generic abstractions as necessary.

Also see: segmented_log, glommio_impl.

In the context of laminarmq this module is intended to provide the storage for individual partitions in a topic.
